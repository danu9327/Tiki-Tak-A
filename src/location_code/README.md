### 상담 후 도움이 필요한 청소년들에게 지원시설로 인도하기 위한 위치정보 처리 코드들

##### 청소년 지원시설 위치 데이터를 공공데이터포털 API로 수집하고 RAG 검색에 적합한 통합 JSON으로 전처리하는 파이프라인입니다.
###### 진짜 공공데이터 포털 사용하기 어렵더라구요 에휴

## 실행순서:
##### python src/location_code/1_fetch_api.py   # API 수집
##### python src/location_code/2_unify.py       # 통합 및 전처리
##### unified_centers_lite.json가 최종 결과

## 1_fetch_api.py 설명
##### .env(절대 지켜!)에 내 API키 있음 "PUBLIC_DATA_API_KEY=발급받은_키"
##### api키로 아래 정보 수집
##### 청소년을 위한 지원시설 위치정보 9개 -> data/rag/location/.json
##### 수집한 api 총 9개
|#|데이터셋|저장 파일|
|-|-------|--------|
|1|청소년디딤센터 위치현황|청소년디딤센터_위치현황.json|
|2|아동청소년보호기관정보 위치현황|아동청소년보호기관정보_위치현황.json|
|3|청소년상담복지센터 위치현황|청소년상담복지센터_위치현황.json|
|4|청소년쉼터 위치현황|청소년쉼터_위치현황.json|
|5|청소년자립지원관 위치현황|청소년자립지원관_위치현황.json|
|6|청소년복지시설관심지점정보 위치현황|청소년복지시설관심지점정보_위치현황.json|
|7|여성·가족·청소년·권익시설정보 위치현황|여성·가족·청소년·권익시설정보_위치현황.json|
|8|청소년성문화센터 위치현황|청소년성문화센터_위치현황.json|
|9|청소년지원시설관심지점 위치현황|청소년지원시설관심지점_위치현황.json|
## 아니 구조? 포맷?이 다양해서 힘들엇음(아래처럼 나뉨)
|유형|해당 API|특징|
|---|--------|----|
|odcloud 형식|1~5번|totalCount, data[]|
|구형 공공데이터 형식|7~9번|response > body > items > item|
|돌연변이 구형|6번|body가 리스트로 한번 더 감싸진 형태|

## 2_unify.py  설명
##### src/location_code/2_unify로 하나의 통합 json으로 전처리
##### (name, address, region, phone, source, extra)로 중구난방인 칼럼을 정리
##### 전화번호 정규화 추가 예시: 020000000->02-000-0000
##### RAG 검색을 위한 text 필드 추가
##### 최종 결과물은 data/rag/location/unified_centers_lite.json이거임

### 아래처럼 싹다 정리
